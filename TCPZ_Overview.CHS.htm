Overview

TCP-Z  V2.6

TCP Window

 

1) Tcpip.sys文件版本。

2) TCP/IP 半开连接数限制数值。从硬盘中的文件 tcpip.sys读取。

3) TCP/IP 半开连接数限制数值。读取自系统核心内存。

4) 指示当前系统的tcpip.sys文件是否是原版文件。
显示为“是”，表示当前文件没有经过任何修改。

5) ID为4226的系统事件统计数量。

6) 峰值记录。左击该图标，将显示历史记录的峰值，及其发生的时间日期。

7) 更改实时数值标签的对齐方式。

8) 已建立的TCP连接数。

9) TCP半开连接数。
半开连接数，是指向正在对外发起的、尚未完成/未收到远程计算机回应的TCP连接。状态标志是SYN-SENT。

10) 新建队列，新建连接速度。 此值读取自tcpip.sys的核心内存。
事实上，Windows是比较这个数值与3内存限制数。

11) 被动的连接。本机已经收到远程的请求，建立连接中。状态标志是SYN-RCVD。
如果一个程序在本机打开一个端口进入监听状态，远程计算机尝试连接此端口时，将产生这个以上的情景。

12) 实时数值/历史峰值。 xx/yy, xx 实时数值，yy是峰值。
连接数图表与速度图表的格式是一致的。

13) 下载速度。千字节/秒。

14) 上传速度。千字节/秒。

15) 网卡数据流量的统计时间。
更改网卡(16)时，此数值将重置为0。

16) 网络连接名称及其对应的网卡名称。

17) 网卡数据流量的统计，D－下载；U － 上传。
更改网卡(16)时，此数值将重置为0。

 

* Windows Server 2003 / 2008 / 7 没有限制，所以图上的限制数值以灰色显示。

 

统计

 

1) 重置为0，重新开始统计。

2) 网卡所发送的数据总流量。
统计周期：自开机后开始。

3)  网卡所接收的数据总流量。
统计周期：自开机后开始。

 

* 如果你正在使用BT/e-mule, 而传入的连接数续读一直为0，那么表示当前的监听端口已经被防火墙拦截；或者你处于内网，而没有正常的影射TCP端口或启用UPNP。

在这种情况下，你会错失很多与远程客户端连接的机会，从而降低下载速度。

 

详情

 

1) 进程启动所用的命令行。

2) 进程启动的日期时间。

3) 物理内存使用量，当前进程的工作集大小。

4) 虚拟内存大小。

5) I/O 读取数据量。
该进程所产生的I/O读取操作数据总量，包括文件、网络、设备I/O等。

6) I/O 写入数据总量.
该进程所产生的I/O写入操作数据总量，包括文件、网络、设备I/O等。

 

补丁

 

1) 修改半开连接数限制值，直接修改核心内存方式。

即使退出TCP-Z，内存中的修改值仍然会保持不变。此修改值在系统关机前一直生效。
系统休眠时，此值保持不变。

重新开机后，需要再次修改限制数。你可以通过安装驱动，或用命令行启动TCP-Z实现自动修改。

tcpz.exe  -limit:200
tcpz.exe  -limit:200  -autoexit

2) 按下此按钮，程序将修改系统限制为你所设定的新限制数。

3) 修改半开连接数限制值，修改文件方式。

TCP-Z的这个功能仅在Windows XP下允许。如果你想以修改文件方式破解Vista，请使用另一工具“通用Tcpip.sys文件补丁—— "Universal Tcpip.sys Patch"。

 

* Windows XP 的Tcpip.sys 文件有4个字节比较，所以最大上限可以达到0xFFFFFFFF.。

* Vista/Windows 7的Tcpip.sys文件只有一个字节比较位，所以最大上限为0xFF (255)。

命令行

tcpz.exe  -limit:200

修改新限制数至200。

tcpz.exe  -limit:200  -autoexit

修改新限制数至200，修改完成自动退出程序。

tcpz.exe  -minimize

运行时窗口最小化。


如果因杀毒软件拦截而造成TCP－Z无法正常启动，你可以通过命令行方式强制不加载驱动，从而使TCP－Z能正常打开窗口。
tcpz.exe  -nodriver

但不加载驱动程序的时候，你将无法使用内存补丁方式破解。

 

设置 tcpz.ini

MaxSpeedGraph=200

图表中速度显示 Y 轴最大值。

NotAdjustSpeedGraph=0

0 - 自动增加 Y 轴上限。
1 - 固定上限数值。
